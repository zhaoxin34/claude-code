# Wolf Fix Workflow 计划

## 背景
用户希望创建一个 `/wolf-fix` 斜杠命令，用于快速修复 Wolf 项目的 bug。用户会通过以下方式提供问题信息：
1. 直接粘贴日志
2. 提供 k8s 命令（如 kubectl logs）或使用 wolf-ops SKILL

## 功能目标
- 完整工作流：问题分析 → 代码定位 → 修复 → 验证
- 触发方式：`/wolf-fix` 斜杠命令

## 实现方案

### 1. SKILL.md 文件结构
创建 `.claude/skills/wolf-fix-workflow/SKILL.md`，包含以下内容：

#### 1.1 工作流程步骤

**步骤 1: 问题分析**
- 解析用户提供的日志或命令输出
- 提取关键错误信息（异常类型、错误消息、堆栈跟踪）
- 识别可能的关键词（类名、方法名、模块名）

**步骤 2: 代码定位**
- 使用 Grep/Glob 搜索相关代码
- 根据堆栈跟踪定位具体文件和代码位置
- 分析问题根因

**步骤 3: 修复方案**
- 提供修复建议
- 使用 Edit 工具进行代码修改

**步骤 4: 验证**
- 运行 `mvn clean compile` 编译检查
- 运行 `mvn test -Dtest=<相关测试类>` 单元测试

### 2. Skill 文件内容模板

```markdown
# wolf-fix-workflow

当用户使用 /wolf-fix 命令或描述了一个 bug 时，执行以下工作流：

## 触发条件
- 用户输入 /wolf-fix
- 用户提供了错误日志或异常堆栈
- 用户描述了问题现象

## 工作流步骤

### 第一步：问题分析
1. 解析用户提供的日志内容
2. 提取关键信息：
   - 异常类型和消息
   - 堆栈跟踪中的关键类名和方法
   - 相关的模块或服务名称

### 第二步：代码定位
1. 使用 Grep 搜索异常类名或关键错误信息
2. 根据堆栈跟踪定位具体代码位置
3. 分析问题根因

### 第三步：修复
1. 提出修复方案
2. 使用 Edit 工具修改代码
3. 确保修改符合项目规范

### 第四步：验证
1. 运行编译检查：`mvn clean compile -pl <模块名>`
2. 运行相关单元测试：`mvn test -Dtest=<测试类名>`

### 第五步：发布
1. （由用户填写发布方法）
2. 可以使用 Bash 执行部署命令

## 注意事项
- 堆栈跟踪中优先关注 "Caused by" 部分
- 涉及多模块时注意模块依赖关系
- 修改代码后运行 /simplify 进行代码简化
```

## 关键文件
- `.claude/skills/wolf-fix-workflow/SKILL.md` - 需要创建的内容

## 验证方式
- 创建完成后，用户可以使用 `/wolf-fix` 命令测试
