# My Secretary Skill 计划

## 背景

用户希望创建一个 Agent Skill，用于管理个人联系人和事件记录。不写代码实现，仅定义技能行为。

## 需求总结

- **数据存储**：SQLite 数据库
- **数据模型**：已定义（见 docs/models.md）
  - contacts 表：联系人信息
  - events 表：事件记录
- **核心功能**：
  1. 智能识别对话中的联系人和事件
  2. 自动记录到 SQLite
  3. 生成丰富的查询报告

## 技能设计

### 技能名称
`my-secretary` 或 `personal-secretary`

### 触发描述
当用户提到联系人、事件、会议、沟通、交流等关键词时触发

### 核心能力

1. **联系人管理**
   - 添加联系人（从对话中提取）
   - 查询联系人信息
   - 更新联系人信息
   - 删除联系人

2. **事件记录**
   - 自动识别事件（从对话中提取）
   - 手动添加事件
   - 查询事件历史
   - 按联系人/时间/类型过滤

3. **智能识别**
   - 从自然语言中提取联系人姓名
   - 判断事件类型（email/chat/phone/meeting 等）
   - 提取事件主题和内容
   - 识别事件时间

4. **报告生成**
   - 联系人统计（按类别：公司/朋友/家人）
   - 事件汇总（本周/本月与某人的互动）
   - 活跃联系人排行
   - 时间线展示

### 数据库操作

- 数据库路径：`./data/my-secretary.db`（相对于项目根目录）
- 启动时自动初始化数据库和表

### 输出格式

- 简洁确认：记录成功时
- 详细报告：查询结果
- 摘要：定期总结

## 实现步骤

1. 创建 SKILL.md 文件
2. 定义触发条件和指令格式
3. 编写 SQLite 操作指南
4. 设计报告模板
5. 创建测试用例
6. 优化描述
