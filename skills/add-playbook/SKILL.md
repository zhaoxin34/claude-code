# 目标

场景1:
我们解决了一个问题，比如安装软件过程中，遇到一个错误，经过一些操作，终于成功了，我们就要记录，怎么导致的问题，如何解决的问题。这样下次如果遇到类似的问题，我们可以利于写的这个问题操作步骤手册，解决问题。

场景2:
操作中有一些操作，必须记录，比如创建了用户，如果不记录，可能以后就会导致忘记用户名和密码，记录下来是非常有意义的。

# 执行逻辑

读取 /Volumes/data/working/my-obsidian/操作记录/ 目录下的文件列表，看看有没有相关主题的问题，如果有，读取这个文档，结合我们要记录的操作步骤，判断是否追加、修改文件。如果没有相同主题的文章，则新建一个文档，记录操作步骤。
