# Git Tag 管理技能

## 概述

这个技能用于创建和管理符合项目规范的 Git tag。

## 触发条件

当用户想要：
- 创建 git tag
- 给 commit 打标签
- 发布版本
- 按照特定版本号规则打 tag

时使用此技能。

## Tag 命名规则

项目使用以下 tag 格式：

```
v{重构版本}.{重大更新版本}.{迭代版本}[.{热修版本}]
```

> 注意：版本号永远是4位

- 重构版本，只有技术架构改变，重构了整个项目后升级这个版本
- 重大更新版本，一般只有发生极其重大的更新才会升级这个版本号
- 迭代版本, 因为项目一般采用敏捷开发，每次开发会设定一个迭代版本，所以每次迭代完成会创建迭代版本号
- 热修版本, 主要是解决迭代版本的问题

## 创建tag的规则

1. 检查项目是否与远程仓库已经同步，如果有未提交的情况，则提示用户提交代码。如果本地没有改动，只是落后或超前，则先同步git仓库。
1. 首先查出目前最新的tag，比如v1.0.0.3，热修版本则变成v1.0.0.4，迭代版本则变成v1.0.1.0
1. 如果用户明确提出本次是版本迭代或热修，则按照用户的要求更新版本号，如果用户不明确，则根据git log确定更新的版本号，比如git log显示做的都是修复bug，则创建热修版本，如果迭代了功能则更新迭代版本号，如果无法判断则更新热修版本号。
1. 如果tag创建成功，则推送至远程仓库
