# 目标

当解决了一个编程问题，如何我们能从中吸收一些经验，或者能复用某些编程的模式，最好的方式是把这个经验写下来，这个技能就是写下这个编程经验。


# 执行逻辑

读取 /Volumes/data/working/my-obsidian/编程经验/
目录下的文件列表，看看有没有相关主题的文档，如果有，读取这个文档，结合我们要记录的内容，判断是否追加、修改文件。如果没有相同主题的文章，则新建一个文档，记录编程经验。

# 文档模板

```
## 标题,  如：spring boot三层架构实施方案

{介绍spring boot三层架构的背景信息, 比如为什么要使用，前提条件，预备知识等等，主要是介绍背景}

## 目标

{介绍要实现哪些目标}

## 方案

{介绍如何实现}

```
